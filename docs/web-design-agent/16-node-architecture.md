# Plan 16: Node Architecture for Web Design Agent Pipeline

> **Purpose:** Define the node-based visualization for each step in the web design agent workflow, enabling visual debugging and understanding of the generation pipeline.

**Dependencies:** All previous plans  
**Estimated Time:** 4-5 hours  
**Parallelizable With:** None

---

## Overview

The web design agent pipeline will be visualized as a series of connected nodes on the canvas. Each node represents a discrete step in the generation process, showing:

- Current status (idle, running, complete, error)
- Input/output data previews
- Controls for regeneration or manual intervention
- Connection points for data flow

---

## Node Component Structure

```typescript
interface WebDesignAgentNode {
	id: string;
	type: WebDesignAgentNodeType;
	status: NodeStatus;

	// Inputs (connected from other nodes)
	inputs: {
		[key: string]: any;
	};

	// Outputs (available to downstream nodes)
	outputs: {
		[key: string]: any;
	};

	// Node-specific config
	config?: {
		autoRetry?: boolean;
		maxRetries?: number;
		slopThreshold?: number;
	};

	// Execution metadata
	executionTime?: number;
	slopScore?: number;
	error?: string;
}

type WebDesignAgentNodeType =
	| 'category-input'
	| 'location-input'
```
